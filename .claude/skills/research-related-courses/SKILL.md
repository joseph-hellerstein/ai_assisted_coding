# Research Related Courses

## Overview

Find and summarize existing courses on AI-assisted coding and related topics to inform the design of new course content. The goal is landscape awareness: what is already taught, at what level, and what is missing.

## When to Use

- Before drafting course objectives, to avoid duplicating existing material
- When evaluating whether a proposed topic or episode is already well-covered elsewhere
- When looking for structural inspiration (episode length, exercise formats, sequencing patterns)

## Core Pattern

1. **Search** for courses using terms like: "AI-assisted coding", "LLM-assisted development", "coding with Claude/GPT/Copilot", "AI pair programming", "prompt engineering for developers"
2. **Check platforms**: Coursera, edX, Udemy, LinkedIn Learning, O'Reilly, DeepLearning.AI, Fast.ai, YouTube playlists, GitHub-hosted curricula
3. **For each relevant course, capture:**
   - Title, platform, author, approximate date
   - Target audience and prerequisites
   - Topic coverage (modules or weeks)
   - What it does well
   - What it omits or treats superficially
4. **Synthesize**: identify the gap this course fills relative to what exists

## Quick Reference

| Signal | What to capture |
|--------|----------------|
| Audience fit | Beginner / intermediate / advanced? Developer or non-developer? |
| Coverage | Specs? Implementation? Code review? Testing? Agents? |
| Format | Video lectures, projects, exercises, live coding? |
| Recency | Post-2023 material is more likely to cover modern LLM tooling |
| Gap | What does this course teach that others don't — or fail to teach well? |

## Common Mistakes

- Listing courses without synthesizing the gap — the output should drive a design decision, not just be a bibliography
- Ignoring free/open resources (GitHub repos, YouTube series, blog-based curricula) which often cover practitioner workflows better than formal courses
